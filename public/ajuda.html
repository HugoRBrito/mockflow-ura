🔍 Visão Geral

Simulador de APIs para testes de integração com o NICE CXone. Permite criar APIs falsas que respondem de forma personalizada baseada nos dados enviados pela URA.

🎯 Como Funciona

1. URA envia request → 2. Sistema aplica MATCH → 3. Retorna resposta configurada

🎬 Cenários

Cada API pode ter múltiplos cenários. O sistema escolhe baseado no MATCH.

// Exemplo: API /clientes
Cenário 1: {"cpf": "111"} → {"status": "VIP"}
Cenário 2: {"cpf": "222"} → {"status": "BLOQUEADO"}
Cenário 3: {} (fallback) → {"status": "NORMAL"}

🔍 Match

Condições para ativar um cenário.

// Por CPF
{"cpf": "12345678901"}

// Por campo do body
{"body.uf": "SP"}

// Por query string
{"query.estado": "RJ"}

// Fallback (qualquer request)
{}

📝 Templates

Use valores do request na resposta.

{
  "cpf_recebido": "{{body.cpf}}",
  "uf_recebida": "{{body.uf}}",
  "protocolo": "PROT-{{timestamp}}"
}

🔒 Validação

Verifica se a URA enviou todos os campos obrigatórios.

// Campos Obrigatórios
["cpf", "nome", "solicitante.nome"]

💬 Dicas

  • Sempre tenha um cenário FALLBACK com match {}
  • Use nomes descritivos nos cenários
  • Teste antes de colocar na URA
